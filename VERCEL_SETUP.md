# Vercel Deployment Guide for Student Performance Predictor

## Prerequisites
- Vercel account (free)
- GitHub repository with your project
- Python 3.9+ and Node.js installed locally

## Option 1: Full Vercel Deployment (Recommended)

### Step 1: Prepare Your Project

```bash
# Install dependencies
npm install

# Train the model locally first
python model_training.py
```

### Step 2: Create `vercel.json`

Already created in your project with configuration for Python and Node.js functions.

### Step 3: Deploy to Vercel

```bash
# Install Vercel CLI
npm install -g vercel

# Login to Vercel
vercel login

# Deploy
vercel
```

Follow the prompts to connect your GitHub repository.

## Option 2: Using Vercel with GitHub

1. Push your project to GitHub
2. Go to https://vercel.com/new
3. Import your GitHub repository
4. Configure settings:
   - Framework: Next.js (or Other)
   - Build Command: Skip (leave empty)
   - Install Command: `pip install -r requirements.txt && npm install`
   - Output Directory: `./public` or `./dist`
5. Click Deploy

## Environment Variables

Add these to Vercel Project Settings → Environment Variables:

```
MODEL_PATH=./saved_model.pkl
PYTHON_VERSION=3.9
```

## Important: Model File

The `saved_model.pkl` file must be committed to your repository. It's generated by:

```bash
python model_training.py
```

## Troubleshooting

### 1. Model Not Found
- Ensure `saved_model.pkl` is in the project root
- Commit it to your GitHub repository
- Add to `.gitignore` if you want to generate it during build

### 2. Python Dependencies
- Verify all packages in `requirements.txt` are compatible with Vercel
- Use `pip freeze > requirements.txt` to generate the file

### 3. Build Timeout
- Keep `saved_model.pkl` pre-built
- Don't train models during deployment

## Alternative: Separate Frontend & Backend

If you want more control, deploy:

1. **Frontend on Vercel** (React/Next.js)
2. **Backend elsewhere** (Railway, Render, or Fly.io)

Create a `.env.local` file:

```
VITE_API_URL=https://your-backend-url.com
```

Then update API calls in your components to use this URL.

## Testing Locally

```bash
# Start Vercel dev environment
vercel dev
```

This will simulate the Vercel environment locally.
